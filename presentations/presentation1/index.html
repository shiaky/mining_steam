E-Ticketsysteme

Allgemeine Funktionsweise

  • Erwerb von Ticket/Guthaben für Ticketmedium
  • Einlesen des Mediums beim Ein- und evtl. Aussteigen
  • Abbuchung beim Ein- oder Aussteigen abhängig vom System

RFID basiert

  • Technologie für berührungslose Sender-Empfänger Systeme
  • Nutzt elektromagnetische Wellen Technik
  • Grundlage vieler gängiger Ticketsysteme
  • 2 Funktionseinheiten: Transponder und Receiver
  • Transponder speichert Daten, z.B NutzerID zur Authentifizierung
  • Receiver liest die Daten auf dem Transponder aus

Smartcard basiert

  • Kunststoffkarte mit integriertem Chip
  • RFID zur Kommunikation zwischen Lesegerät und Karte
  • Meist mit Guthaben genutzt
  • Speichern von z.B. Jahres- oder Seniorentickets möglich
  • Optional: Registrierung bei Betreiber für Komfortfunktionen
  • Beispiele: Oyster Card London, Octopus Card Hong Kong

NFC basiert

  • Basiert auf RFID Technik
  • 3 Modi:
    • Reader: Lesegerät für RFID Geräte
    • Card Emulation: emuliert Smartcard
    • Peer to Peer: Kommunikation zwischen 2 NFC Geräten
  • Beispiele: Pay-As-You-Go, Touch & Travel der DB

Empfehlungen für eine privacy-sensible Praxis

Grundsätze bei der Entwicklung

  • beratender statt sanktionierendem Datenschutz
  • Wenn eine Person nicht agiert, sollen von ihr auch keine personenbezogenen Daten erfasst werden.
  • Die Interessen aller Beteiligten sollten einbezogen werden, ohne jedoch die Funktionalität zu beeinträchtigen.
  • Die Entwicklung von Anfang bis Ende unter dem Aspekt der Datensicherheit betrachten
  • Visibility and Transparency als Grundsatz u Prozesse zu testen

Schutzziele

  • Verfügbarkeit: gesichter Zugriff in festgelegten Zeiten.
  • Vertraulichkeit: Das System garantiert sicheren Zugriff
  • Integrität: Die Informationen sind echt gesichert.
  • Transparenz: Sammeln der Daten soll nachvollzogen, überprüft und bewertet werden können.
  • Nichtverkettbarkeit: Daten werden so getrennt wie möglich gehalten
  • Intervenierbarkeit: Eine Person kann von ihren Rechten gebrauch machen.
### E-Ticketsysteme * Stammdatenerhebung nötig? → freiwillige Angabe empfohlen * **Nichtverkettbarkeit garantieren:** Stamm- und Zahlungsdaten, Fahrverhalten * **Transparenz schaffen:** Daten für Fahrgast einseh- und löschbar machen * **Empfehlung für Fahrgast:** Aufladung nur Bar → keine Zuordnung von Zahlungsmitteln

Alle Daten werden lokal gespeichert

Pro

  • Informationen können offline abgerufen werden
  • Man müsste sich lokal einen Virus einfangen, damit Daten entwendet werden können

Kontra

  • Da Daten offline gespeichert werden, kann es zu veralteten Datensätzen kommen
  • Vorallem ältere Smartphones/PCs können mit den Daten überfordert sein

Daten werden teileweise lokal gespeichert

Pro

  • Daten immer up to date
  • Schneller wenn das Verkehrsnetz groß ist (vorallem bei älteren Smartphones)

Kontra

  • Man muss dem Serverbetreiber vertrauen, dass er die Daten nicht speichert
  • Durch eine geknackte Verschlüsselung ist es trotzdem möglich an die Daten ranzukommen
  • Anfällig für gesetzliche Backdoors (z.B. NSA)
